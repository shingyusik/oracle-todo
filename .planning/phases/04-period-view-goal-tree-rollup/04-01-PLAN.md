<objective>
Build the store-agnostic core of the period view: the shared `PeriodView`/`GoalNode` nested type, the `period_view(horizon, period)` service method, the in-memory `assemble()` tree-build (visited-set + depth-cap + anomaly walk per SC3/D-08/D-09), and the InMemory store loader that composes the existing `list_items` read primitive (D-11 InMemory half). Prove VIEW-03/VIEW-04/SC1/SC2/SC3 against the InMemory store with unit/integration tests.

Purpose: This is the shared transform both stores feed in Phase 4 — the type and the `assemble()` walk are identical code for InMemory and Persistent, so building and locking them first means the Wave 2 SQL CTE loader only has to produce the same flat working set. The parity test (Wave 3) ultimately validates only that the loaders agree because the assembler is shared.
Output: `PeriodView`/`GoalNode` types + `period_view()` + `assemble()` + InMemory loader in `queries.rs`; `MAX_GOAL_DEPTH` promoted to `pub(super)` in `goal.rs`; new `tests/integration/period_view.rs` registered in `tests/integration.rs` with in-memory behavior tests.
</objective>
