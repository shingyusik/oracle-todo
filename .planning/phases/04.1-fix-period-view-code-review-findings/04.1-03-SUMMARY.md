## Decisions Made
- **A1 fallback applied (Rust 2024 E0364):** `pub use goal::MAX_GOAL_DEPTH;` does not compile — a private/`pub(super)` item cannot be re-exported as `pub`. The plan's documented fallback (`#[cfg(test)]` `pub const` accessor) was refined: the integration test crate links the *non*-`cfg(test)` library build, so a `#[cfg(test)]` gate would be invisible to it. Used an ungated `pub const MAX_GOAL_DEPTH = goal::MAX_GOAL_DEPTH;` accessor — still single-sourced (binds the production value, no re-mirrored `64`); widened surface is a single non-sensitive integer cap (threat T-04.1-05: accept).
- **Open Question 1 → option (b):** loader-level comparison. The persistent raw `goal->task->goal` view is compared against an in-memory `goal->task` view built via the validating API (G2 not created, provably unreachable in the persistent store under D-01). No production InMemory test-only seeding hook added; the `pub(super)` `ServiceStore` boundary is unchanged.
