<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shell ŌåÆ CLI process (clap args) | Untrusted positional/flag input (`--horizon`, `--scheduled`, `--period`, dates, `--parent-id`) crosses into the process and is forwarded as raw strings to the service. |
| CLI adapter ŌåÆ TodoService | The adapter calls service methods; ALL validation, state-machine, and audit logic lives behind this boundary (CORE-01/CORE-03). The adapter sets no status. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-CLI-01 | Tampering | `--horizon` / `--scheduled` / `--period` / date args | mitigate | Forward raw strings to `propose_goal` / `agenda` / `date_range`; the service's `Horizon::from_str`, `validate_goal_anchor`, and `parse_day` reject junk with `TodoError::Validation` ŌćÆ exit 2. Adapter does no parsing except `period`'s `Horizon::from_str`, which maps `Err(String)` to `TodoError::Validation` (same exit 2). |
| T-05-CLI-02 | Elevation of Privilege | `goal propose` actor / status | mitigate | Adapter NEVER sets status; `actor` defaults to `agent` (`default_value = "agent"`). The service state machine sets `proposed` for the agent actor ŌćÆ approval gating cannot be bypassed from the CLI (SC4). |
| T-05-CLI-03 | Denial of Service | `period` view over cyclic/over-depth `parent_id` legacy data | accept | Already mitigated in `queries.rs` (visited set + `MAX_GOAL_DEPTH` cap, `anomaly_count`, never errors). This plan only exposes the existing read; it adds no new traversal. |
| T-05-CLI-04 | Tampering | `update --parent-id` to a non-Goal / terminal parent | mitigate | Routed through `update_item` ŌåÆ `ensure_relation(.., Goal, ..)`; non-Goal/terminal parent ŌćÆ Policy error ŌćÆ exit 2. Adapter adds no bespoke validation (D-07). |
| T-05-CLI-SC | Tampering | package installs | accept | This phase installs NO new packages (RESEARCH ┬¦Standard Stack: "no new dependencies"). No Package Legitimacy Audit required; no install task exists. |
</threat_model>
