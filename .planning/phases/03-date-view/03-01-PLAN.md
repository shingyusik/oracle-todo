<objective>
Add the two pure, side-effect-free date-view query methods to `TodoService` in `application/service/queries.rs`: a single-date `agenda` (`scheduled == D OR due == D` union, deduped by id) and an arbitrary `[from, to]` `date_range` (scheduled-only). Both return a flat `Vec<TodoItem>` in a single deterministic order. This is the cheaper flat half of Core Value, computed in the service so CLI and API will agree (CORE-03 / D-01a parity guard).

Purpose: VIEW-02 (date grouping + unscheduled bucket) and VIEW-05 (scheduled+due agenda) require a shared service-layer read primitive. The deterministic sort IS the CLI/API parity guarantee — there is no `DateView` struct (D-01); grouping and the unscheduled bucket are the adapter's job in Phase 5.
Output: `agenda` and `date_range` methods plus the private `iso_day`, `sort_date_view`, and `open_tasks` helpers in `queries.rs`. No new files, no schema change, no adapter code.
</objective>
